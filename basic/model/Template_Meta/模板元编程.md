## 概念

**模板元编程**（Template Metaprogramming, TMP）是在编译期通过模板机制进行代码生成与计算的技术，利用编译器的模板实例化机制，在编译期执行代码逻辑，提高性能和灵活性。

## 优点

| 优点 | 说明 |
|:---|:---|
| **提高代码的可重复性与泛化能力** | 一次编写，多处使用 |
| **减小运行时开销** | 计算在编译期完成 |
| **实现类型安全的高级抽象** | 编译期类型检查 |

## 基础

- **模板特化**
- **递归模板**

## 示例1：编译期计算阶乘

### 代码实现

```cpp
template <int N>
struct Factorial {
    // 改为 inline static const int value = ... 就不需要类外定义，来防止取内存崩
    static const int value = N * Factorial<N - 1>::value;
};

// 特化基础情况
template <>
struct Factorial<0> {
    static const int value = 1;
};

// 为 Factorial<0>::value 提供定义，使其可以取地址
template <int N>
const int Factorial<N>::value;

template <>
const int Factorial<0>::value;  // 静态编译会报错，但不影响运行
```

### 使用

```cpp
// 编译期计算，运行时直接使用结果
constexpr int result = Factorial<5>::value;  // result = 120
```

### 工作原理

```
Factorial<5>::value
= 5 * Factorial<4>::value
= 5 * (4 * Factorial<3>::value)
= 5 * (4 * (3 * Factorial<2>::value))
= 5 * (4 * (3 * (2 * Factorial<1>::value)))
= 5 * (4 * (3 * (2 * (1 * Factorial<0>::value))))
= 5 * (4 * (3 * (2 * (1 * 1))))
= 120
```

## 示例2：判断类型是否可相加

### 代码实现

```cpp
#include <type_traits>

// 默认情况：不可相加
template <typename T, typename = void>
struct is_addable : std::false_type {};

// 特化：可以相加
template <typename T>
struct is_addable<T,
    decltype(void(std::declval<T>() + std::declval<T>()))
> : std::true_type {};
```

### 关键技术

**std::declval**：不创建实际对象，但返回一个对应类型的右值引用

```cpp
// declval 的作用
// std::declval<T>() 返回 T&&
// 用于在不创建对象的情况下，模拟调用成员函数或操作符
```

### 使用

```cpp
static_assert(is_addable<int>::value, "int should be addable");
static_assert(!is_addable<void>::value, "void should not be addable");
```

## 示例3：类型列表与索引访问

### 类型列表定义

```cpp
template <typename... Ts>
struct typelist {};
```

### 按索引获取类型

```cpp
template <typename List, std::size_t N>
struct TypeAt {};

// 基础情况：索引为 0
template <typename Head, typename... Tail>
struct TypeAt<typelist<Head, Tail...>, 0> {
    using type = Head;
};

// 递归情况：索引不为 0
template <typename Head, typename... Tail, std::size_t N>
struct TypeAt<typelist<Head, Tail...>, N> {
    using type = typename TypeAt<typelist<Tail...>, N - 1>::type;
};
```

### 使用

```cpp
using list = typelist<int, double, char>;
using third = TypeAt<list, 1>::type;  // third = double

// 输出类型名称
std::cout << typeid(third).name() << std::endl;
```

### 工作原理

```
typelist<int, double, char>

TypeAt<list, 0>::type  → int
TypeAt<list, 1>::type  → double
TypeAt<list, 2>::type  → char
```

## 高级技巧

### constexpr 变量（C++11）

```cpp
// 使用 constexpr 替代 static const
template <int N>
struct Factorial {
    static constexpr int value = N * Factorial<N - 1>::value;
};

template <>
struct Factorial<0> {
    static constexpr int value = 1;
};

// 不需要类外定义，可以直接取地址
constexpr auto val = Factorial<5>::value;  // 完全合法
```

### inline 变量（C++17）

```cpp
// C++17 引入 inline 变量
template <int N>
struct Factorial {
    inline static const int value = N * Factorial<N - 1>::value;
};

// 也不需要类外定义
```

### 变参模板 + 折叠表达式（C++17）

```cpp
#include <iostream>

// 打印所有参数
template <typename... Args>
void print_all(const Args &...args) {
    ((std::cout << args << " "), ...);  // C++17 折叠表达式
}

// 求和
template <typename... Args>
auto sum_all(const Args &...args) {
    return (args + ...);  // C++17 折叠表达式
}

// 使用
// print_all(1, 2.5, "hello");
// sum_all(1, 2, 3, 4);  // 返回 10
```

## 常用类型特性

```cpp
#include <type_traits>

// 判断类型
std::is_integral<T>::value      // 是否整数类型
std::is_floating_point<T>::value // 是否浮点类型
std::is_pointer<T>::value       // 是否指针类型
std::is_reference<T>::value     // 是否引用类型
std::is_const<T>::value         // 是否 const 类型
std::is_same<T, U>::value       // T 和 U 是否相同类型

// 类型转换
std::remove_reference_t<T>      // 移除引用
std::remove_const_t<T>          // 移除 const
std::add_pointer_t<T>           // 添加指针
std::make_unsigned_t<T>         // 转换为无符号类型

// 类型操作
std::enable_if<condition, T>::type  // 条件满足时类型为 T，否则无 type 成员
std::decay_t<T>                     // 衰退类型（数组转指针，函数转函数指针，移除 cv）
```

## 静态断言

```cpp
// 编译期断言
static_assert(condition, "error message");

// 示例
template <typename T>
void check() {
    static_assert(std::is_integral<T>::value, "T must be integral");
}

// check<int>();   // OK
// check<double>(); // 编译错误：T must be integral
```

## 总结

| 技术版本 | 特性 | 说明 |
|:---|:---|:---|
| **C++98** | 模板特化、递归模板 | 基础 TMP |
| **C++11** | constexpr、变参模板、type_traits | 增强 TMP |
| **C++14** | constexpr 函数放宽 | 更灵活的编译期计算 |
| **C++17** | 折叠表达式、if constexpr | 简化参数包处理 |
| **C++20** | Concept | 更清晰的约束 |

### 编译期 vs 运行期

| 特性 | 编译期 | 运行期 |
|:---|:---|:---|
| **计算时间** | 编译时 | 运行时 |
| **性能开销** | 无 | 有 |
| **灵活性** | 受限 | 高 |
| **调试难度** | 高 | 低 |
