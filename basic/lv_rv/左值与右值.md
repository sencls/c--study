## 定义

| 类型 | 说明 | 示例 |
|:---|:---|:---|
| **lvalue (左值)** | 具有持久存储的对象，可出现在赋值语句左侧，可被取地址 | 变量名、引用 |
| **rvalue (右值)** | 临时对象或没有持久存储的值，通常出现在赋值语句右侧，不能被取地址 | 字面量、临时对象、表达式结果 |

### 基本示例

```cpp
int a = 10;
//  ^    ^
//  左值  右值

int b = a;      // a 是左值，但作为表达式结果时是右值
int c = a + b;  // a + b 的结果是右值
```

## C++11 细分

在 C++11 中，右值被进一步细分为：

### 分类

| 类型 | 英文 | 说明 | 示例 |
|:---|:---|:---|:---|
| **纯右值** | prvalue | 临时对象、字面量 | `10`, `3.14`, `"hello"` |
| **将亡值** | xvalue | 表示即将被移动的对象 | `std::move()` 的结果 |

### 价值类别

```
         表达式
           |
    ┌──────┴──────┐
    |             |
  左值         右值
 (lvalue)     (rvalue)
    |        ┌────┴────┐
    |        |         |
   左值     纯右值    将亡值
          (prvalue) (xvalue)
```

## 类型判断

### 判断函数

```cpp
#include <type_traits>

std::is_lvalue_reference<decltype()>::value  // 判断是否是左值引用
std::is_lvalue_reference_v<decltype()>        // 简写版本 (C++17)

std::is_rvalue_reference<decltype()>::value  // 判断是否是右值引用
std::is_rvalue_reference_v<decltype()>        // 简写版本 (C++17)
```

### 重要：判断左值时的 decltype

```cpp
// 判断左值时，需要使用两层括号
int a = 10;
std::is_lvalue_reference_v<decltype((a))>  // true
std::is_lvalue_reference_v<decltype(a)>    // false

int &b = a;
std::is_lvalue_reference_v<decltype(b)>    // true
```

### 完整示例

```cpp
int a = 10;
int &b = a;
int &&c = std::move(a);

// 判断类型
std::cout << std::is_lvalue_reference_v<decltype((a))> << std::endl;  // true
std::cout << std::is_lvalue_reference_v<decltype(b)> << std::endl;    // true
std::cout << std::is_rvalue_reference_v<decltype(c)> << std::endl;    // true
```

## 模板万能引用（转发引用）

### 概念

**万能引用**（Universal Reference）或 **转发引用**（Forwarding Reference）是 C++11 引入的引用类型。

### 特点

1. **表现形式为 `T&&`**：T 是模板参数
2. **编译器自动推导**：解析为左值引用还是右值引用
3. **统一处理**：能统一处理左值与右值

### 推导规则

| T 的推导结果 | 引用类型 |
|:---|:---|
| 普通类型 | 右值引用 |
| 引用类型 | 左值引用 |

### 示例

```cpp
template <typename T>
void fun1(T &&x) {
    std::cout << "T is lv reference: " << std::is_lvalue_reference_v<T> << std::endl;
    std::cout << "T is rv reference: " << std::is_rvalue_reference_v<T> << std::endl;
    std::cout << "x is lv reference: " << std::is_lvalue_reference_v<decltype(x)> << std::endl;
    std::cout << "x is rv reference: " << std::is_rvalue_reference_v<decltype(x)> << std::endl;
}

int a = 10;
fun1(a);              // T 推导为 int&，x 为 int&
fun1(std::move(a));   // T 推导为 int，x 为 int&&
```

## 引用折叠规则

处理引用类型的嵌套。

### 规则

| 组合 | 结果 | 说明 |
|:---|:---:|:---|
| `& &` | `&` | 左值引用 |
| `&& &` | `&` | 左值引用 |
| `& &&` | `&` | 左值引用 |
| `&& &&` | `&&` | 右值引用 |

**总结**：只要有一个是左值引用，结果就是左值引用；只有两个都是右值引用，结果才是右值引用。

### 示例

```cpp
typedef int& T1;
typedef int&& T2;

T1& a;   // int&  → &
T1&& b;  // int&  → &
T2& c;   // int&  → &
T2&& d;  // int&& → &&
```

### 重要：右值引用作为表达式

```cpp
// 右值引用类型做表达式时 → 左值
template <typename T>
void fun(T &&x) {
    // x 本身是左值（有名字）
    // std::is_lvalue_reference_v<decltype(x)> == true
}
```

## std::forward 完美转发

### 目的

在模板函数中，将接收到的参数以其原有的值类别（左值、右值）传递给另一个函数。

### 重载函数示例

```cpp
void process(int &x) {
    std::cout << "(int&)" << std::endl;
}

void process(int &&x) {
    std::cout << "(int&&)" << std::endl;
}

// 错误的转发
template <typename T>
void wrapper_bad(T &&x) {
    process(x);  // x 总是左值（有名字），总是调用 process(int&)
}

// 正确的转发
template <typename T>
void wrapper_good(T &&x) {
    process(std::forward<T>(x));  // 保持原有的值类别
}

// 使用
int a = 10;
wrapper_good(a);              // 调用 process(int&)
wrapper_good(std::move(a));   // 调用 process(int&&)
```

### 完整示例

```cpp
template <typename T>
void display_type(T &&param) {
    std::cout << "T is lv reference: " << std::is_lvalue_reference_v<T> << std::endl;
    std::cout << "param is lvalue reference: " << std::is_lvalue_reference_v<decltype(param)> << std::endl;
    std::cout << "T traits normal type is int: " << std::is_integral_v<typename std::remove_reference_t<T>> << std::endl;
}
```

## 类型推导规则

模板编程中，编译器根据传入的实参自动推导模板参数类型。

### 规则

1. **模板参数与函数参数匹配时**，自动推导
2. **引用的处理**：引用折叠
3. **cv 限定符的去除**：const 与 volatile 修饰会去除

### 移除引用

```cpp
#include <type_traits>

template <typename T>
void remove_ref_example() {
    using Type = std::remove_reference_t<T>;
    // Type 是 T 去除引用后的类型
}

// 示例
// std::remove_reference_t<int&>    → int
// std::remove_reference_t<int&&>   → int
// std::remove_reference_t<int>     → int
```

## 原样转发

### 目的

在模板函数中，将接收到的参数以其原有的值类别（左值、右值）传递给另一个函数，确保泛型代码可以正确处理参数。

### 目的

1. **保持参数值类别不变**，确保正确调用函数重载
2. **利用移动语义**，避免不必要的拷贝，提升性能
3. **编写通用、复用性强的代码**

### 实现

通过 **模板万能引用** + **std::forward<T>()**

### 示例对比

#### 版本1：值传递（丢失引用）

```cpp
template <typename F, typename T1, typename T2>
void flip1(F f, T1 t1, T2 t2) {
    f(t2, t1);  // t1, t2 是拷贝，丢失引用
}

void ftemp(int v1, int &v2) {
    std::cout << v1 << ' ' << ++v2 << std::endl;
}

void use_ftemp1() {
    int i = 100, j = 99;
    flip1(ftemp, j, 42);
    std::cout << j << std::endl;  // j 仍然是 99，没有被修改
}
```

#### 版本2：万能引用（保留引用）

```cpp
template <typename F, typename T1, typename T2>
void flip2(F f, T1 &&t1, T2 &&t2) {
    f(t2, t1);  // t1, t2 保持原有引用
}

void use_ftemp2() {
    int i = 100, j = 99;
    flip2(ftemp, j, 42);
    std::cout << j << std::endl;  // j 被修改为 100
}
```

#### 版本3：完美转发（处理右值）

```cpp
void gtemp(int &&i, int &j) {
    std::cout << i << ' ' << ++j << std::endl;
}

void use_ftemp3() {
    int i = 100, j = 99;
    flip2(gtemp, i, j);  // 需要完美转发才能正确处理右值
    std::cout << j << std::endl;
}
```

## 总结

### 值类别总结

```
表达式
  ├── 左值 (lvalue)
  │   └── 有名字、有持久存储
  │       变量、解引用、*ptr
  │
  └── 右值 (rvalue)
      ├── 纯右值 (prvalue)
      │   └── 临时对象、字面量
      │       10, 3.14, "hello"
      │
      └── 将亡值 (xvalue)
          └── std::move() 的结果
```

### 引用类型

| 引用类型 | 声明 | 绑定对象 |
|:---|:---:|:---|
| 左值引用 | `T&` | 左值 |
| 右值引用 | `T&&` | 右值 |
| 万能引用 | `T&&` (T 是模板参数) | 左值或右值 |

### 完美转发三要素

1. **模板参数 T**：用于推导原始类型
2. **万能引用 T&&**：接收任意值类别
3. **std::forward\<T>()**：保持原始值类别转发

```cpp
template <typename T>
void perfect_forward(T &&arg) {
    other_function(std::forward<T>(arg));
}
```
