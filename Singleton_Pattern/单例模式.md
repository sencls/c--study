# 单例模式

> 本文件整理了 `singleton_pattern.cpp` 中的注释内容

---

## 概念

**单例模式**（Singleton Pattern）是创建型设计模式，确保一个类在整个程序的生命周期中有且只有一个实例，并提供一个全局访问点来获取该实例。

---

## 作用

| 作用 | 说明 |
|:---|:---|
| **控制实例数量** | 确保只有一个，防止创建多个而导致的资源浪费或状态不一致，例如：数据库连接池、配置管理类等 |
| **提供全局访问点** | 使得程序在任何位置都可以方便的访问实例，对于需要在多模块或组件之间共享的资源或服务尤为重要 |
| **延迟实例化** | 通常采用懒加载模式，即第一次需要才创建，有助于节省系统资源，特别是在实例创建成本较高或初期并不需要该实例的情况下 |
| **避免命名冲突** | 通过单例实例作为一个类的静态成员，避免在全局命名空间中引入多个实例，减少命名冲突的风险 |
| **管理共享资源** | 在多线程模式下，单例可以有效管理共享资源，确保线程安全 |

---

## 使用场景

### 适用场景

1. **需要确保全局只有一个实例**
   - 配置管理
   - 日志系统
   - 设备驱动

2. **需要全局访问点来协调系统中的多个部分**
   - 缓存
   - 线程池

## 实现方式

### 方式1：局部静态变量方式（推荐）

#### 特点

- C++98 时存在隐患（多线程时可能生成多个）
- C++11 编译器优化，避免了隐患（静态成员多次调用只会创建一次）
- 最简洁、最安全的实现方式

#### 代码

```cpp
class Single2 {
private:
    Single2() {}
    Single2(const Single2 &) = delete;
    Single2 &operator=(const Single2 &) = delete;

public:
    static Single2 &GetInst() {
        static Single2 single;  // 第一次调用生成对象，后续调用共用
        return single;
    }

    ~Single2() {
        std::cout << "destory\n";
    }
};

// 使用
auto &instance = Single2::GetInst();
```

#### 优点

- ✅ 线程安全（C++11 及以上）
- ✅ 自动销毁
- ✅ 懒加载
- ✅ 代码简洁

---

### 方式2：指针方式（饿汉式）

#### 特点

- 程序开始时就创建单例
- 需要在对应的 .cpp 文件中或 main() 函数中初始化

#### 代码

```cpp
class Single2Hungry {
private:
    Single2Hungry() {}
    Single2Hungry(const Single2Hungry &) = delete;
    Single2Hungry &operator=(const Single2Hungry &) = delete;
    static Single2Hungry *single;

public:
    static Single2Hungry* GetInst() {
        if (single == nullptr) {
            single = new Single2Hungry();
        }
        return single;
    }

    ~Single2Hungry() {
        std::cout << "destory\n";
    }
};

// 在 .cpp 文件中初始化
Single2Hungry* Single2Hungry::single = Single2Hungry::GetInst();
```

#### 缺点

- ❌ 析构存在问题（内存泄漏）
- ❌ 不是懒加载
- ❌ 初始化顺序不确定

---

### 方式3：指针方式（懒汉式）+ 锁

#### 特点

- 调用时才创建
- 使用互斥锁保证线程安全
- 存在性能问题（每次调用都需要加锁）

#### 代码

```cpp
#include <mutex>

class Single2Lazy {
private:
    Single2Lazy() {}
    Single2Lazy(const Single2Lazy &) = delete;
    Single2Lazy &operator=(const Single2Lazy &) = delete;
    static Single2Lazy *single;
    static std::mutex _mtx;

public:
    static Single2Lazy *GetInst() {
        std::lock_guard<std::mutex> lock(_mtx);  // 存在串行的可能
        if (single == nullptr) {
            single = new Single2Lazy();
        }
        return single;
    }

    ~Single2Lazy() {
        std::cout << "destory\n";
    }
};

// 在 .cpp 文件中初始化
Single2Lazy *Single2Lazy::single = nullptr;
std::mutex Single2Lazy::_mtx;
```

#### 缺点

- ❌ 性能差（每次调用都加锁）
- ❌ 存在串行的可能

---

### 方式4：双重检查锁定（DCLP）

#### 特点

- 减少锁的竞争
- 第一次检查时不加锁
- 仍然存在线程不安全问题

#### 代码

```cpp
static Single2Lazy *GetInst() {
    if (single != nullptr) {  // 第一次检查，不加锁
        return single;
    }
    _mtx.lock();
    if (single != nullptr) {  // 第二次检查，加锁后
        _mtx.unlock();
        single = new Single2Lazy();
    }
    single = new Single2Lazy();
    _mtx.unlock();
    return single;
}
```

#### 缺点

- ❌ **仍然存在线程不安全问题**
  - 因为 `new` 的关系，在多线程时可能取到开辟了地址但没有构造的结果
  - 导致数据混乱或者直接崩溃

---

### 方式5：C++11 once_flag（推荐）

#### 特点

- 使用 `std::call_once` 保证线程安全
- 使用 `std::shared_ptr` 管理生命周期
- 最优雅的多线程单例实现之一

#### 代码

```cpp
#include <mutex>
#include <memory>

class singleOnceFlag {
private:
    singleOnceFlag() {}
    static std::shared_ptr<singleOnceFlag> _instance;

public:
    ~singleOnceFlag() {
        std::cout << "desorty\n";
    }
    singleOnceFlag &operator=(const singleOnceFlag &) = delete;
    singleOnceFlag(const singleOnceFlag &) = delete;

    static std::shared_ptr<singleOnceFlag> GetInst() {
        static std::once_flag flag;
        std::call_once(flag, []() {
            _instance = std::shared_ptr<singleOnceFlag>(new singleOnceFlag());
        });
        return _instance;
        // 为什么不用 make_shared() -> 要调用构造函数，但是私有无法使用
    }
};

// 在 .cpp 文件中初始化
// 注意静态成员的类外初始化，链接时可能出错
std::shared_ptr<singleOnceFlag> singleOnceFlag::_instance = nullptr;
```

#### 优点

- ✅ 线程安全
- ✅ 懒加载
- ✅ 使用智能指针管理内存
- ✅ 优雅的 C++11 风格

#### 为什么不用 make_shared？

```cpp
// 不能使用 make_shared 的原因
// _instance = std::make_shared<singleOnceFlag>();  // 错误！

// 原因：
// make_shared 需要调用构造函数
// 但构造函数是私有的
// make_shared 不是友元函数，无法访问私有构造函数
```

## CRTP 单例模板

### 概念

**CRTP**（Curiously Recurring Template Pattern，奇异递归模板模式）

将派生类作为模板参数传递给基类，即一个类继承自一个以自身为模板参数的基类。

### 常用场景

- 静态多态
- 接口的默认实现
- 编译时策略选择
- **单例模板基类**

### 基本结构

```cpp
template<typename T>
class Singleton {
protected:
    Singleton() {}
    Singleton(const Singleton &) = delete;
    Singleton &operator=(const Singleton &) = delete;

public:
    static T &GetInst() {
        static T instance;
        return instance;
    }
};

// 使用 CRTP
class RealClass : public Singleton<RealClass> {
    // RealClass 的构造函数需要是 protected 或 public
    // 因为 Singleton<RealClass> 需要调用它
protected:
    RealClass() {}

public:
    void doSomething() {
        std::cout << "RealClass doing something" << std::endl;
    }
};

// 使用
auto &instance = RealClass::GetInst();
instance.doSomething();
```

### 完整示例

```cpp
template<typename T>
class SingletonBase {
protected:
    SingletonBase() = default;
    SingletonBase(const SingletonBase &) = delete;
    SingletonBase &operator=(const SingletonBase &) = delete;

public:
    static T &GetInstance() {
        static T instance;
        return instance;
    }
};

class MySingleton : public SingletonBase<MySingleton> {
    friend class SingletonBase<MySingleton>;

private:
    MySingleton() {
        std::cout << "MySingleton constructed" << std::endl;
    }

public:
    void DoWork() {
        std::cout << "Working..." << std::endl;
    }
};

// 使用
int main() {
    MySingleton &singleton = MySingleton::GetInstance();
    singleton.DoWork();
    return 0;
}
```

## 友元类与私有析构

### 极端单例实现

通过使用友元类来调用私有的析构函数，实现更极端的单例。

```cpp
class Singleton {
private:
    Singleton() {}
    ~Singleton() {
        std::cout << "Singleton destroyed" << std::endl;
    }
    Singleton(const Singleton &) = delete;
    Singleton &operator=(const Singleton &) = delete;

    // 声明友元类
    friend class SingletonDeleter;

public:
    static Singleton &GetInst() {
        static Singleton instance;
        return instance;
    }
};

// 专门用于删除单例的友元类
class SingletonDeleter {
public:
    void Destroy() {
        // 可以访问 Singleton 的私有析构函数
    }
};
```

### 推荐实现

#### 简单场景

```cpp
class Singleton {
private:
    Singleton() = default;
    Singleton(const Singleton &) = delete;
    Singleton &operator=(const Singleton &) = delete;

public:
    static Singleton &GetInst() {
        static Singleton instance;
        return instance;
    }
};
```

#### 复杂场景（需要控制析构时机）

```cpp
class Singleton {
private:
    Singleton() = default;
    Singleton(const Singleton &) = delete;
    Singleton &operator=(const Singleton &) = delete;

public:
    static std::shared_ptr<Singleton> GetInst() {
        static std::once_flag flag;
        static std::shared_ptr<Singleton> instance;
        std::call_once(flag, []() {
            instance = std::shared_ptr<Singleton>(new Singleton());
        });
        return instance;
    }
};
```
